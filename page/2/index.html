Learn Pytorch

Pytorch入门

Tensor（张量）

  • Tensors 类似于 NumPy 的 ndarrays ，同时 Tensors 可以使用 GPU 进行计算。
  • 任何使张量会发生变化的操作都有一个前缀 ‘_’，如x.add_(y)
  • 改变一个 tensor 的大小或者形状，你可以使用 torch.view
  • 一个元素 tensor ，使用 .item() 来获得这个 value

Pytorch自动微分

Tensor

  • torch.Tensor 是包的核心类。如果将其属性 .requires_grad 设置为 True，则会开始跟踪针对 tensor 的所有操作。完成计算后，您可以调用 .backward() 来自动计算所有梯度。该张量的梯度将累积到 .grad 属性中。
  • 要停止 tensor 历史记录的跟踪，您可以调用 .detach()，它将其与计算历史记录分离，并防止将来的计算被跟踪。
  • 要停止跟踪历史记录（和使用内存），您还可以将代码块使用 with torch.no_grad(): 包装起来。在评估模型时，这是特别有用，因为模型在训练阶段具有 requires_grad = True 的可训练参数有利于调参，但在评估阶段我们不需要梯度。
  • 还有一个类对于 autograd 实现非常重要那就是 Function。Tensor 和 Function 互相连接并构建一个非循环图，它保存整个完整的计算过程的历史信息。每个张量都有一个 .grad_fn 属性保存着创建了张量的 Function 的引用，（如果用户自己创建张量，则 grad_fn 是 None ）。
  • 如果你想计算导数，你可以调用 Tensor.backward()。如果 Tensor 是标量（即它包含一个元素数据），则不需要指定任何参数backward()，但是如果它有更多元素，则需要指定一个gradient 参数来指定张量的形状。

Pytorch神经网络

现在对于自动梯度(autograd)有一些了解，神经网络是基于自动梯度 (autograd)来定义一些模型。一个 nn.Module 包括层和一个方法 forward(input) 它会返回输出(output)。

一个典型的神经网络训练过程包括以下几点：

  1. 定义一个包含可训练参数的神经网络
  2. 迭代整个输入
  3. 通过神经网络处理输入
  4. 计算损失(loss)
  5. 反向传播梯度到神经网络的参数
  6. 更新网络的参数，典型的用一个简单的更新方法：weight = weight - learning_rate *gradient

PyTorch图像分类器

通常来说，当你处理图像，文本，语音或者视频数据时，你可以使用标准 python 包将数据加载成 numpy 数组格式，然后将这个数组转换成 torch.Tensor

  • 对于图像，可以用 Pillow，OpenCV
  • 对于语音，可以用 scipy，librosa
  • 对于文本，可以直接用 Python 或 Cython 基础数据加载模块，或者用 NLTK 和 SpaCy

Learn Git

引言

​ 之前一直在用Git，但是也仅限于简单的Clone、Push等操作，没有充分发挥其版本控制的优秀特性。因此决定抽出半天时间，对Git做一个相对完整的了解。

集中式vs分布式

集中式

​ 版本库是集中存放在中央服务器的，而干活的时候，用的都是自己的电脑，所以要先从中央服务器取得最新的版本，然后开始干活，干完活了，再把自己的活推送给中央服务器。中央服务器就好比是一个图书馆，你要改一本书，必须先从图书馆借出来，然后回到家自己改，改完了，再放回图书馆。

分布式

​ 首先，分布式版本控制系统根本没有“中央服务器”，每个人的电脑上都是一个完整的版本库，这样，你工作的时候，就不需要联网了，因为版本库就在你自己的电脑上。既然每个人电脑上都有一个完整的版本库，那多个人如何协作呢？比方说你在自己电脑上改了文件A，你的同事也在他的电脑上改了文件A，这时，你们俩之间只需把各自的修改推送给对方，就可以互相看到对方的修改了。

分支管理

创建与合并分支

  • 几个概念：分支指针（指向提交）、HEAD指针（指向分支指针）

解决冲突

  • 当Git无法自动合并分支时，就必须首先解决冲突。解决冲突后，再提交，合并完成。

  • 解决冲突就是把Git合并失败的文件手动编辑为我们希望的内容，再提交。

  • 用git log --graph --pretty=oneline --abbrev-commit命令可以看到分支合并图。

分支管理策略

  • 通常，合并分支时，如果可能，Git会用Fast forward模式，但这种模式下，删除分支后，会丢掉分支信息。

  • 如果要强制禁用Fast forward模式 (–no-ff)，Git就会在merge时生成一个新的commit，这样，从分支历史上就可以看出分支信息。

  • 在实际开发中，分支管理的几个基本原则：

    • 首先，master分支应该是非常稳定的，也就是仅用来发布新版本，平时不能在上面干活；
    • 那在哪干活呢？干活都在dev分支上，也就是说，dev分支是不稳定的，到某个时候，比如1.0版本发布时，再把dev分支合并到master上，在master分支发布1.0版本；
    • 你和你的小伙伴们每个人都在dev分支上干活，每个人都有自己的分支，时不时地往dev分支上合并就可以了。

Bug分支

Feature分支

  • 开发一个新feature，最好新建一个分支；

  • 如果要丢弃一个没有被合并过的分支，可以通过git branch -D <name>强行删除。

多人协作

推送分支

  • master分支是主分支，因此要时刻与远程同步；
  • dev分支是开发分支，团队所有成员都需要在上面工作，所以也需要与远程同步；
  • bug分支只用于在本地修复bug，就没必要推到远程了，除非老板要看看你每周到底修复了几个bug；
  • feature分支是否推到远程，取决于你是否和你的小伙伴合作在上面开发。

抓取分支

多人协作模式

因此，多人协作的工作模式通常是这样：

  1. 首先，可以试图用git push origin <branch-name>推送自己的修改；
  2. 如果推送失败，则因为远程分支比你的本地更新，需要先用git pull试图合并；
  3. 如果合并有冲突，则解决冲突，并在本地提交；
  4. 没有冲突或者解决掉冲突后，再用git push origin <branch-name>推送就能成功！

如果git pull提示no tracking information，则说明本地分支和远程分支的链接关系没有创建，用命令git branch --set-upstream-to <branch-name> origin/<branch-name>。

系统性风险衡量方法（Volatility, correlation and tails for systemic risk）

  • 系统性风险：By Daniel Tarullo, the Board of Governors of Federal Reserve

    • 如果公司未能履行对债权人和客户的义务，将对金融系统和更广泛的经济产生重大不利后果，金融机构就具有系统重要性。
    • 在这个定义中，一家具有系统重要性的公司未能履行义务是造成系统性风险的原因。衡量系统性风险的指标与公司破产或濒临破产有关，这是高杠杆公司股票估值下降的必然后果。
      • 杠杆：将借到的货币追加到用于投资的现有资金上
    • 当一家金融公司的股本价值跌至其未偿负债的足够小一部分时，该公司就无法运作。在景气时期，这样的公司很可能被收购，或许能够筹集新的资本，或者可能面临有序的破产。
    • 如果这种资本短缺恰好发生在金融部门已经财政紧张的时候，那么政府就面临着一个问题，即在其他途径不再可用的情况下，是否要用纳税人的钱来拯救公司。这种资本短缺对实体经济是有害的，因为这家公司的倒闭将对整个金融和实体部门产生影响。因此，如果一家公司可能在金融部门本身陷入困境时面临巨大的资本缺口，那么它就是具有系统风险的。

  • 衡量具有系统重要性的企业的资本短缺情况，以便为公司编制系统性风险指数

    • 一家公司的资本缺口取决于其杠杆率和此类危机可能导致的股本损失。虽然杠杆程度是可以衡量的，但危机中的股权损失必须是可以预测的 。
    • 在使用杠杆和MES（ Marginal Expected Shortfall ）计算的危机中，公司的资本缺口决定了我们的系统风险指数（SRISK）。
    • SRISK最高的公司是在危机中对市场资本不足贡献最大的公司，因此是系统风险最大的公司。从概念上讲，这一计算类似于定期应用于金融公司的压力测试，但在这里，它只使用公开的信息进行，而且快速且成本低廉。

  • SRISK指数的计算方法

    • 为了构建计算该指数所需的MES（边际预期损失）预测，我们提出了一个市场和公司股票收益动态的条件异方差双变量模型。
    • 该模型从时变的波动性、相关性和尾部三个方面描述了收益率的行为。
    • 该规范没有对过程做出具体的分布假设。采用基于GARCH和DCC的多步建模方法对波动性和相关性进行拟合。对结果的推断基于灵活的方法，并允许潜在的非线性依赖。
    • 该模型用于构建提前一个周期和多个周期的MES预测，分别称为短期MES预测和长期MES预测。
      • 短期和长期MES的定义分别是以一天内2%的市场跌幅和六个月内40%的市场跌幅为条件。
      • 在短期情况下，使用波动性、相关性预测和非参数尾部期望估计量可以很容易地构建预测。
      • 长期MES预测不能以封闭的形式获得，但基于模拟的方法会产生这样的预测。长期预测是在构建SRISK指数时使用的预测。

  • SRISK指数测试方法

    • 对2000年7月至2010年7月期间94家美国顶级金融公司的系统风险贡献进行了抽样分析。
    • 金融公司被分成4类：存托机构、保险公司、经纪交易商和标记为其他的剩余类别，这一类别还包括非存款机构和房地产相关公司。
    • 我们首先使用我们的GARCH/DCC/非参数方法来研究面板的样本内动力学。然后，我们进行两种不同的预测练习。我们首先执行一个短期MES预测应用程序，并使用不同的标准将我们方法的预测能力与两个基准进行比较。然后，我们进行了一项长期的MES和SRISK预测工作，我们检查了危机不同阶段产生的系统性风险机构的排名，以及样本中总SRISK的动态。

  • lSRISK测试结果

    • 该模型表明，MES最重要的决定因素是时变的公司波动性和与市场的相关性。

    • MES对于波动较大、市场多元化程度不高的公司来说很高。

    • 时间序列分析表明，以历史标准衡量，金融危机期间的MES水平是最高的。

    • 然而，MES行业排名在时间上一直保持稳定，经纪交易商和其他行业是受影响最大的行业。

    • 短期MES预测应用表明，相对于基准，我们的MES预测能够很好地预测风险。

    • 长期MES预测结果显示，SRISK对金融危机不同阶段的系统风险公司进行了有用的排名。在雷曼破产前一年半，SRISK排名前十的公司中有八家被证明是陷入困境的机构。

    • 结果还记录了从2007年7月开始金融系统资本化的恶化情况，截至2010年7月，金融系统似乎尚未完全恢复。

  • 财务平衡模型

    • 上述方法关键假设是指在金融系统陷入困境期间，个别公司的资本短缺会给实体经济带来外部成本。
    • 这些成本首先是提供担保的纳税人的成本。然而，它们还包括在整个金融部门受到资本限制时特别严重的外部性。当经济低迷时，企业的破产无法被更强大的竞争对手吸收，债务将蔓延到整个金融和实体经济，因为金融部门的自然功能将受到限制。
    • 当该系统资本不足时，它将不再为普通的日常业务提供信贷，经济将陷入停滞。
    • 因此，资本短缺对一家公司及其债券持有人来说是危险的，但如果它恰好发生在该行业其余部分资本不足的时候，对经济来说就是危险的。
    • 请注意，企业的目标函数没有考虑到它在危机中产生的负外部性成本。因此，在没有监管干预的情况下，公司可能会过度杠杆化。例如，当收益的波动性低时，风险就低，最优杠杆率就高。

  • 衡量每家公司在危机中的资本缺口是有意义的。如果资产价值与权益的审慎比率为k，则公司在第1期末的资本缓冲为：
    $$
    k(B_1F_{i1}+G_{i1}+W_{i1})
    $$

  • 当这一价值为正值时，公司被认为经历了资本短缺。

  • 然后，我们估计危机中的预期股本损失，并计算由此产生的资本缺口。我们将危机定义为市场指数跌破某一阈值C，我们称之为短暂的“系统性事件”。则在T2时刻的预期资金短缺估计为：

  • MES（Marginal Expected Shortfall）表示某一单个银行对金融系统 ES（Expected Shortfall）的边际贡献

  • 波动率(Volatility) $\sigma=\sqrt{\frac{\sum_{i=1}^{n}{(x_i-\overline{x})^2}}{n-1}}$

  • 风险值VaR（Value at Risk）在a%的置信度水平下，投资机构在未来N天内的可能损失不超过X，这被称为VaR。

  • 即便用了VaR来预估预期损失，但在预估是假设资产波动呈现正态分布，然而为了因应**厚尾(fat tail)**的情况，也就有Expected Shotfall的诞生，或者有另一个常听到的名称为 Conditional VaR。

  • ES的概念其实很好理解，在给定的信心水准(confidence level)和一段期间(T)下，去计算可能产生的预期损失。从上面的示意图可以发现，其实在尾端未必如常态分佈机率逐渐减少，而是可能又有一个峰的存在，而ES就是要算出尾端风险的期望值。

  • $ES=\mu+\sigma\frac{e^{-Y^2/2}}{\sqrt{2\pi}(1-X)}$

  • 公式中的 Y 和 X 关係可以理解为，Y 为在标准常态分佈下，(1−X)%信心水准下的临界值，举例而言，X为1%，则为即为99%信心水准的临界值，即为-2.326。

  • 边际贡献率=（边际贡献/销售收入）×100%；

    • 如果是一种产品，则：边际贡献率=（单位边际贡献/单价）×100%；

    • 如果是多种产品，则：边际贡献率=∑（每种产品的边际贡献率×该产品销售收入占全部销售收入的比重）。边际贡献率是指边际贡献在销售收入中所占的百分比。通常，边际贡献率是指产品边际贡献率，可以理解为每一元销售收入是边际贡献所占的比重，它反映产品给企业做出贡献的能力。

  • capital shortfall

  • 其中Ri2和Rm2是第2期的公司和市场(预测)收益，MES代表边际预期缺口（ Marginal Expected Shortfall），公司收益的尾部预期条件是市场处于左尾。

  • 如果金融系统出现巨大的资本缺口CS（capital shortfall），将会引发一场真正的危机。资本缺口最大的公司是这场危机的最大贡献者，也是被认为具有最大系统风险的机构。因此，危机中预计会出现的大规模资本短缺不仅反映了单个公司的脆弱性，也反映了系统性风险。

  • 我们将机构I的系统性风险指数定义为$SRISK_i=min(0, CS_{i0})$

  • 百分比形式$SRISK%_i=SRISK_i/\sum_i{SRISK_i}$
